Foi apresentado um fluxo básico sobre o uso do git/github:

  • Repositório para manter o código
  • Relação entre mudanças na máquina local e no repositório (remoto ou origin) no Github
  • Principais comandos: git add, git commit -m "..", git push origin <branch>
  • Commits na branch principal (master), por ora...

Uso básico do Git/Github

Cada repositório possui uma ou mais branchs, que nada mais são que fluxos diferentes do código, sendo uma forma de isolar o desenvolvimento de funcionalidades. No sentido de que ao invés de desenvolver direto na master, cria-se uma branch alternativa a partir de um ponto, implementa o que for necessário - sem a interferência de possíveis alterações de outros desenvolvedores e sem inserir uma versão instável na branch principal.

Pull Request

Uma forma de inserir na branch principal as nossas mudanças é fazer o merge da nossa branch na master por meio do Pull Request. Trata-se de uma funcionalidade do github em que buscamos validar com o restante da equipe as nossas mudanças antes de mergear. Este processo é conhecido como Code Review.
